Quickstart#

This page provides a minimal and reproducible FGManip workflow. The goal is to help you run one complete iteration from data preparation to policy evaluation.

Start Here#

  • Prerequisite: complete Installation

  • Expected outcome: one validated asset/task sample, one runnable training command, and one evaluation report

1. Data: Build One Sample Fast#

1.1 URDF Interaction Pose: annotate or download#

You can start from either of the following:

  • Annotate your own asset with the FGManip interaction pose / grasp pose annotation tool.

  • Download curated FGManip asset data from the official FGManip release channel (xxx) to quickly run baseline experiments.

This dual path supports both rapid prototyping and custom asset onboarding.

1.2 Collect and merge trajectories#

# record
python record.py --env-id grasp_part --skill-type grasp_part \
  --object-name bottle --part-name cap \
  -n 10 --record-dir demos --save-video

# merge (typically only successful trajectories)
python utils/merge_trajectory.py -i demos/grasp_part -o demos/grasp_part/trajectory.h5 -p trajectory.h5

1.3 Replay and convert (optional)#

# replay with target observation/control mode
python utils/replay_trajectory.py \
  --traj-path demos/trajectory.h5 --use-first-env-state \
  -c pd_joint_delta_pos -o rgb --save-traj --num-envs 10 -b physx_cpu

# convert to LeRobot format (if needed)
python utils/convert_to_lerobot.py \
  --traj-path demos/trajectory.h5 \
  --output-dir demos/lerobot_grasp_part \
  --task-name "Grasp Part" --fps 30 --robot-type panda

Done check: trajectory can be replayed; interaction behavior is correct; conversion script completes without errors.

2. Simulation Interactive Demo#

Before large-scale data generation or training, validate that your asset annotation and interaction settings are physically correct.

  • Target part can be reached and manipulated consistently.

  • Contact behavior is stable (no obvious penetration / severe jitter).

  • Task success condition can be triggered from valid trajectories.

Use examples and utilities in the FGManip repo to quickly sanity-check interaction quality.

3. Training#

FGManip provides examples adapted to multiple policy backends. Choose one backend based on your experiment target and data format.

  • ACT

  • DP3

  • OpenVLA-OFT

  • Lerobot Pi05

  • StarVLA GR00T

Each backend page includes setup, data requirements, training/evaluation commands, and troubleshooting notes.

4. Evaluation#

We recommend reporting results from three complementary dimensions:

Behavior#

  • Task success rate

  • Completion stability and failure type statistics

Understanding#

  • Instruction / target selection correctness

  • Part-level intent execution consistency

Perception#

  • Robustness across view, occlusion, and appearance variations

  • Sensitivity to observation mode and visual quality settings

Next Steps#

  • Scale data collection with more assets and tasks.

  • Compare multiple policy backends under the same task protocol.

  • Track reproducible benchmarks for future leaderboard integration.
